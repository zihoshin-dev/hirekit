## 1. 현재 상태 진단

### 1.1 아키텍처 현황

| 구성요소 | 현재 상태 | 평가 |
|---------|----------|------|
| **코어 엔진** | Python CLI (typer+rich), 14개 데이터소스, 5차원 스코어카드 | 기반은 탄탄하나 분석 깊이 얕음 |
| **데이터** | 79개 기업 JSON (demo data), v2_reports (79개 MD+JSON) | 정적 스냅샷, 검증 메커니즘 없음 |
| **웹 (GitHub Pages)** | index.html + demo.html + tools.html + ko.html + en.html | 정적 HTML, 인터랙션 제한적 |
| **LLM 파이프라인** | OpenAI/Anthropic/Ollama 어댑터, 섹션별 프롬프트 | 단일 패스, 멀티에이전트 아님 |
| **MCP 서버** | stdio JSON-RPC, 5개 tool 노출 | 기본 수준 |
| **테스트** | engine/core/sources/cli/llm 테스트 존재 | 커버리지 확인 필요 |

### 1.2 핵심 문제점 (Critical Gaps)

#### 데이터 신뢰성 위기
- 79개 기업 데이터가 **정적 JSON 스냅샷** — 수집 시점 이후 검증/갱신 메커니즘 없음
- `company_comparator.py`, `career_strategy.py`, `interview_prep.py`에 **하드코딩된 기업 정보** (tech stack, culture hints, company data) — 실제 수집 데이터와 이원화
- `meta.json`의 스코어카드 점수가 evidence 없이 주관적으로 부여된 기업 다수

#### 분석 깊이 부족
- 12섹션 리포트지만 실제 채워지는 섹션은 소스 가용성에 따라 3-5개
- 재무분석: DART 기본 재무제표만 (매출/영업이익 성장률) — 동종업계 비교, 밸류에이션, 투자 라운드, 번레이트 등 없음
- 조직문화: 커뮤니티 리뷰 건수 기반 스코어링 — 실질적 문화 인사이트 없음
- 기술 분석: GitHub 공개 레포 스코어만 — 실제 기술 스택 깊이, 기술 부채, 엔지니어링 문화 분석 없음

#### 사용자 여정 단절
- CLI 도구 — 개발자만 접근 가능, 비개발 직군 (PM, 디자인, 마케팅 등) 진입 장벽 높음
- 데모 사이트: 기업 선택 → 스코어카드 표시 → 끝. **"그래서 나는 뭘 해야 하지?"** 에 대한 답 없음
- 이력서/JD/자소서/면접 기능이 각각 독립 — **통합 커리어 전략 엔진** 부재

#### 웹 사이트 한계
- GitHub Pages 정적 사이트 — 사용자 인터랙션 최소 (기업 선택, 스코어 표시 수준)
- tools.html: JD 분석기, 이력서 점수, 자소서 클리셰, 기업 비교 — 모두 **클라이언트 사이드 로직만** — 실제 분석 없이 UI 목업에 가까움
- 반응형이지만 모바일 최적화 미흡

### 3.2 국민연금 데이터 — HireKit의 "Hidden Moat"

**기존 플랫폼 어디에도 없는 독점 인사이트**:

```
신규취득자 - 상실가입자 = 순채용속도 (net hiring velocity)
상실 급증 = 구조조정/대량퇴사 시그널
기여금 / 9% x 100 = 추정 월급 (+-15-20%)
```

- 월별 트래킹으로 **실시간 고용 건강 지표** 생성
- DART 연간 직원수와 **교차검증** → confidence 상승
- 20%+ YoY 가입자 증가 기업 = 강력한 채용 시그널
- 데이터 지연: ~6주, 규모대 데이터 (5이하/10이하/50이하/100이하/1000이하/1000이상)

### 3.3 재무분석 심층화

기존: 매출/영업이익 성장률만 → 보강:

| 분석 항목 | 산출 방식 | 근거 |
|----------|----------|------|
| **PER/PBR/ROE/ROA** | DART 재무제표 자동 계산 | 기본 밸류에이션 |
| **CEO Pay Ratio** | 임원보수/평균연봉 (DART 임원개인보수) | 보상 형평성 지표 |
| **부채비율 + 현금흐름** | 재무제표 3개년 추이 | 재무 안정성 |
| **동종업계 Peer 비교** | 업종코드 기반 자동 매핑 | 상대적 포지셔닝 |
| **스타트업 생존 예측** | Gradient Boosting (재무+텍스트 퓨전) | ArXiv:2409.03668 — +2.3pp 정확도 |
| **비재무 안정성 프록시** | CEO 배경, 문화 트렌드, 채용/해고 시그널 | Tandfonline 2024 (n=520) |

> 핵심: 단순 재무 비율(Altman Z-Score 등) 대신 **ML 기반 복합 예측**이 일관되게 우월 (MDPI JRFM 2025 체계적 리뷰)

### 3.4 데이터 검증 프레임워크

```python
# 각 기업 JSON에 _meta 필드 표준화
{
  "_meta": {
    "collected_at": "2026-03-22T10:00:00+09:00",
    "verified_at": "2026-03-22T10:05:00+09:00",
    "confidence": 0.85,        # 0.0-1.0
    "source_urls": [...],
    "staleness_days": 0,
    "cross_validated": true,   # 2+ 소스 교차검증 여부
    "version": 2
  }
}
```

- 79개 기업 일괄 검증 스크립트 (`tools/verify_company_data.py`)
- 교차검증 엔진: 동일 데이터포인트를 2개 이상 소스에서 수집 → 불일치 시 confidence 하락
- 자동 갱신: DART/GitHub/News RSS 주기적 delta 업데이트

## 4. 분석 엔진 혁신

### 4.1 스킬 매칭 엔진 재설계

**현재**: `tech_taxonomy.py` 키워드 매칭 (~65% 정확도)

**목표**: 하이브리드 파이프라인 (~87% 정확도)

```
JD 텍스트 → LLM 스킬 추출 (NLP4HR 2024 방식)
         → ESCO/KECO 정규화
         → SBERT 시맨틱 임베딩 (ko-sroberta-multitask)
         → 가중 복합 스코어 (w1 x 스킬매칭 + w2 x 시맨틱 + w3 x 경력 + w4 x 학력)
```

| 매칭 방식 | 정확도 | 비용 | 비고 |
|----------|--------|------|------|
| 키워드 TF-IDF (현재) | ~65% | 최저 | 동의어 누락 |
| SBERT 시맨틱 | ~82% | 낮음 | 80MB 로컬 실행 |
| 하이브리드 파이프라인 (목표) | ~87% | 중간 | NER + ESCO + SBERT + 가중합 |
| Zero-shot LLM | ~85% | 높음 | 건당 10-50x 비용 |

핵심 기술 선택:
- **ESCO v1.2.0** (13,890개 스킬, CC BY 4.0) + **KECO** (한국표준직업분류) 매핑
- **LLM in-context learning** for 스킬 추출 (BIO 태깅 F1 67% → LLM-ICL 85%+, NLP4HR 2024)
- **Skill-LLM** 방식: fine-tuning 없이 구조화 프롬프트로 즉시 배포 (ArXiv:2410.12052)
- 한국어 임베딩: `jhgan/ko-sroberta-multitask` 또는 `upstage/solar-embedding-1-large`

### 4.2 조직문화 분석 혁신

**현재**: 커뮤니티 리뷰 "건수" 기반 스코어

**보강**:

| 분석 레이어 | 기법 | 근거 |
|------------|------|------|
| **감성분석 6차원** | ABSA (급여/워라밸/상사/문화/성장/복지) | NLP4HR 2024, ScienceDirect 2025 |
| **시계열 트렌드** | 리뷰 감성 7개년 추이 → 개선/악화 판정 | MDPI Electronics 2025 (n=1,500) |
| **전직자 가중** | 전직자 부정 리뷰에 높은 정보 가중치 | STM 연구 (n=67,529, FAANG) |
| **잡플래닛 인증 리뷰** | 고용보험 인증 리뷰 우선 반영 | 잡플래닛 2024.6 정책 |
| **NPS 퇴사율 추정** | 월별 상실가입자 비율 → 연간 이직률 | 국민연금 데이터 |

### 4.3 스코어카드 가중치 재설계

2030세대 설문 결과 반영 (잡코리아 2025, n=1,252):

| 차원 | 기존 가중치 | 보강 가중치 | 근거 |
|------|-----------|-----------|------|
| Job Fit | 30% | 25% | - |
| Growth | 20% | 20% | - |
| Compensation | 15% | 15% | - |
| Culture Fit | 15% | **25%** | 50.3% WLB 1위, 34.5% 비윤리 리더십 거부 |
| Career Leverage | 20% | 15% | - |

신규 서브차원 추가:
- **WLB 점수** (NPS 퇴사율 + 리뷰 ABSA)
- **리더십 윤리 점수** (경영진 리뷰 ABSA + 뉴스 감성)
- **원격근무 가능성** (채용공고 분석 + 리뷰)

### 4.4 복합 스코어링 공식

```
Company Score = Σ(wi x Di x Ci)

Di = 차원별 점수 (0-5)
wi = 가중치
Ci = 신뢰도 계수 (소스 수, 교차검증 여부에 따라 0.5-1.0)
```

신뢰도 표시 정책:
- n < 50 리뷰: Wilson Score Interval 적용
- 소스 1개만: Bayesian Shrinkage (모집단 평균 방향으로 수축)
- **절대 confidence bounds 없이 점수 표시하지 않음**

## 5. 커리어 전략 엔진

### 5.1 커리어 경로 예측

**LABOR-LLM** 방식 (Stanford GSB/HAI, ArXiv:2406.17972):
- 이력서 텍스트 기반 fine-tuned 7B 모델이 대형 범용 LLM보다 다음 직무 예측에서 우월
- 핵심: 한국어 직무명 의미 보존 (숫자 ID 인코딩 시 성능 유의미 하락)

**NAOMI 듀얼뷰** 접근:
- BERT 시맨틱 임베딩 + 커리어 전이 그래프 토폴로지 결합
- 한국판: 사람인/잡코리아 전이 데이터 + BERT-Korean 직무명 임베딩

**Karrierewege** 참고 (ArXiv:2412.14612):
- 500K+ 커리어 패스 + ESCO 매핑 → 한국판: 워크넷 + KECO 매핑으로 구축

### 5.2 이력서 AI 리라이팅 파이프라인

Resume-Matcher (26.4K GitHub stars) 참고:

```
마스터 이력서 (1회 작성)
    → JD 수신 시 AI 테일러링 (LiteLLM 추상화)
    → ESCO 정규화 → 스킬 갭 분석
    → 60+ 프롬프트 멀티패스 리라이팅
    → ATS 최적화 자동 적용
    → Before/After 차이점 하이라이팅
```

- **LiteLLM** 추상화로 Ollama(로컬) <-> Claude(클라우드) 무중단 전환
- 파이프라인 깊이(60+ 단계)가 모델 크기보다 중요

### 5.3 경력기술서 분석 (신규)

## 7. 멀티에이전트 AI 아키텍처

### 7.1 3-Agent Debate + Graph of Thoughts

```
+--------------------------------------------------+
|          Graph of Thoughts Orchestrator            |
|  (복합 분석 브랜치 분기 -> 평가 -> 병합 -> 합성)     |
+----------+----------+----------+-----------------+
| Agent 1  | Agent 2  | Agent 3  | Meta-Critic     |
| 재무     | 문화/조직 | 기술/성장 | 교차검증+편향감지 |
| 분석가   | 분석가   | 분석가   |                  |
+----------+----------+----------+-----------------+
|           Multi-Agent Debate Layer                 |
|  (각 에이전트가 서로의 평가를 비판적으로 검토)       |
|  -> Echo Chamber 방지: 서로 다른 모델 백본 사용     |
+--------------------------------------------------+
```

- **Graph of Thoughts** (AAAI 2024): ToT 대비 +62% 품질, -31% 비용
- **Multi-Agent Debate** (ICML 2024): 사실성 + 추론 유의미한 개선
- **Echo Chamber 방지** (NeurIPS 2024): 에이전트별 다른 모델/프롬프트
- **R.A.I.S.E. 순차 추론** (ArXiv:2504.12090): 시장→재무→팀→성장 체인

### 7.2 LightRAG 기업 지식 그래프

```
DART 공시 + 뉴스 + 기술블로그 + 리뷰
    | LLM 엔티티/관계 추출
    v
Knowledge Graph (Neo4j/PostgreSQL)
    | Dual-level Retrieval
    v
"토스의 최근 3년 채용 방향성과 핵심 기술 투자는?"
```

- Microsoft GraphRAG 대비 **10x 토큰 절감** (EMNLP 2025, 29.9K GitHub stars)
- 기업별 Knowledge Graph → 복합 질의 응답
- 백엔드: Neo4j, PostgreSQL, MongoDB, OpenSearch 중 선택

### 7.3 면접 시뮬레이터

```
사용자 프로필 + 목표 기업 데이터
    → 면접관 AI 에이전트 생성
    → 기업 문화/직무 맞춤 질문 출제
    → 사용자 답변 입력
    → 실시간 피드백 (STAR 구조, 키워드, 구체성)
    → 종합 점수 + 개선 포인트
```

- 1000+ 질문 뱅크 (기업별/직군별/연차별/카테고리별)
- 카테고리: 인성/기술/상황/문화핏/압박/역질문/케이스스터디
- STAR 답변 자동 생성 (사용자 경력 기반)
